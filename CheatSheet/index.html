CheatSheet Git

  • Verbindung VS Code
  • Upload von Projekten
  • Alphabetische Übersicht von Befehlen

Verbindung von VS Code zu git

  1. VS Code öffnen
  2. Terminal -> New Terminal
  3. Eingabe im Terminal: git --version
    • Wird eine Version angezeigt, ist Git bereits installiert
    • Ist noch keine Version installiert, öffnet sich ein Pop-up-Fenster zur Installation
  4. Eingabe: git config --global user.name "username"
    • Hier wird als "username" der jew. Username des zu verbindenden Github Profils angegeben
  5. Eingabe: git config --global user.email "e-mail adresse"
    • Hier wird als "e-mail address" die mit dem Github Profil verbundene E-Mail-Adresse angegeben
  6. Eingabe: git config --global --list
    • Mit diesem Befehl können die zuvor angegebenen Daten noch einmal abgerufen und geprüft werden

Upload von Projekten in Git/GitHub

  1. Repository erstellen
    • In Github über das "+" "New repository" auswählen
    • Im sich öffnenden Fenster einen Namen vergeben und das repository erstellen
    • Es öffnet sich eine neue Ansicht mit Optionen zum Übertragen. Hier kopiert man sich die 3 Zeilen unter dem Punkt "...or push an existing repository from the command line"
  2. Im VS Code das zu übertragende Projekt öffnen
  3. Terminal öffnen
  4. Eingabe: git init
    • Zur Initialisierung der Übertragung in ein bestehendes Repository
  5. Eingabe: git add .
    • Hinzufügen aller zugehörigen Files des ausgewählten Projekts - Achtung: alles was aktuell in VS Code geöffnet ist, wird zusammengefasst und als ein Projekt übertragen!
  6. Eingabe: git commit -m "Name für den Upload vergeben"
    • Vergabe eines Namens für den aktuellen Upload - im Falle von Aktualisierungen eines bereits existierenden Projekts ggf. angeben, was angepasst wurde
  7. Einfügen der kopierten Zeilen aus Github
    • git remote add origin https://github.com/"username"/"Name Repository".git
      • Fügt ein Remote-Repository hinzu
    • git branch -M main
      • Definiert die Branch des Uploads
    • git push -u origin main
      • Überträgt schließlich die lokalen Daten in das Repository auf Github
  8. Ggf. Username und PAT eingeben, um die Verbindung zu autorisieren
  9. Sobald der Upload erfolgt ist, wird dies im Terminal angezeigt und nach refresh der Github Seite ist das Repository ebenfalls online zu sehen
